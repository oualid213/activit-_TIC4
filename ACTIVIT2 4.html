  • techniques de l’information et de la communication (TIC)

    Les technologies de l'information et de la communication (TIC) rassemblent les moyens nécessaires pour la manipulation, la transmission et l'échange d'informations. Elles résultent de la fusion de l'informatique, des télécommunications, de l'électronique et de l'audiovisuel. Les TIC favorisent la communication, l'accès à l'information, le stockage, la manipulation et la création de contenu en utilisant des technologies telles que l'informatique, Internet, l'audiovisuel et les télécommunications. des pages accessibles sur des sites.

2_ Catégories et objectifs

  1. .Informatique en réseau : échange de données numériques entre systèmes informatiques.
  2. Télécommunications : communication qui n'est pas nécessairement informatique
  3. Électronique grand public : composants et équipements tels que télévisions et lecteurs DVD.

  4. Objectifs visés :

    • Connectivité : faciliter la communication entre ordinateurs utilisant différents logiciels.
    • Modularité : utiliser un ensemble restreint d'appareils polyvalents.
    • Simplicité de déploiement : solution générale facile à installer dans diverses configurations
    • Facilité d'utilisation : outils de communication simplifiant l'interaction avec le réseau.
    • Fiabilité : détection et correction des erreurs

3_Composants de TIC

  1. La partie matérielle

    • a. Portable, TV, téléphone mobile, tablette interactive, caméra numérique, robot...
    • b. Poste de travail : Une machine capable de transmettre des données vers les réseaux (PC, MAC, terminal SUN X, ...), serveurs...

  2. Composants logiciels

    • a. Systèmes d'exploitation, logiciels d'application et d'utilisation des divers composants matériels.
    • b.Réseaux internet ou mobiles.

Caractéristiques

  1. Rapidité d'exécution et déplacements réduits.
  2. Décentralisation du pouvoir et performance accrue.
  3. Meilleur accès à l'information pour tous.
  4. Partage confidentiel entre niveaux hiérarchiques.
  5. Expansion mondiale grâce au réseau d'information.
  6. Circulation d'information de qualité entre stations
  7. Transmission d'informations en tout lieu et temps

Initiation aux techniques de communication

  • Objectifs de communication

    1. Objectif cognitif : Attirer l'attention, accroître la notoriété et fournir des informations.
    2. Objectif affectif : Modifier ou renforcer l'image de produit et influencer les attitudes et opinions
    3. Objectif comportemental : Modifier les comportements en incitant à l'achat ou à l'essai

  • Système de communication :

    Un système de communication comprend généralement :

    • Émetteur : transforme le message en signaux.
    • Récepteur : reçoit et reconstruit le message
    • Message : codé par l'émetteur
    • Codage : traduction structurée du message
    • Décodage : interprétation du récepteur
    • Réponse : opinion du destinataire.
    • ....

  • Types de communication

    1. Communication institutionnelle (corporate) : Mettre en valeur l'image de marque par la publicité, le parrainage, etc.
    2. Communication commerciale ou de produit : Mettre en évidence les caractéristiques et avantages du produit de manière crédible.

  • Schémas de communication

  • Interpersonnelle : Message entre un émetteur et un collaborateur avec un éventuel retour.
  • De groupe : Message à plusieurs destinataires, comme les salariés ou lors de réunions, avec possibilité de retour et de divers avis.
  • De masse : Publicité sans retour direct, mesurée par des critères tels que les ventes.

  • Média et Multimédia :

    1. Média : Moyen impersonnel pour diffuser des informations (presse, radio, télévision, cinéma, Internet).
    2. Multimédia : Utilisation interactive de divers modes de représentation de l'information

  • Réseau informatique : ensemble d'ordinateurs et de terminaux autonomes connectés pour échanger des informations numériques dans un domaine géographique défini.
  • Mise en réseau : utilisation d'outils et de tâches pour relier les ordinateurs et permettre le partage des ressources.
  • Un réseau est composé d'équipements interconnectés permettant la communication de données entre applications, indépendamment des distances qui les séparent.
  • Évolution des réseaux : initialement utilisés pour la communication entre terminaux distants et un site central, ils intègrent désormais les données, la parole et la vidéo.

      • des notions fondamentales d'un réseau :

    1. Interconnexion : assure la transmission des données entre les nœuds.
    2. Communication : permet l'échange de données entre les processus (programmes en cours d'exécution).
    3. Nœud (node) : extrémité d'une connexion dans un réseau.
    4. Processus : programme en cours d'exécution, représentant le point de communication dans un réseau informatique.

2/ Les caractéristiques de base d’un réseau sont :

  1. .La topologie
  2. Le débit exprimé en bits/s (ou bps)
  3. La distance maximale (ou portée)
  4. Le nombre de nœuds maximum

3/ Quelques terminologies en réseau :

  1. Station de travail
  2. Noeud
  3. Paquet
  4. Réseaux Hétérogènes

Éléments d'un reseau :

  1. Les ordinateurs
  2. les logiciels
  3. Les supports du LAN
  4. Les équipements d'interconnexion du LAN

5/ Classification des réseaux :

Les réseaux peuvent être classifiés en fonction de différents critères :

  • Par portée l’envergure (La distance):

    1. LAN (Local Area Network = réseau local d'entreprise) : appelé réseau local, constitué d'ordinateurs et de périphériques reliés entre eux et implantés (planté) dans une même entreprise.
    2. MAN : (Métropolitan Area Network = Réseau métropolitain ou urbain) : Correspond à la réunion de plusieurs réseaux locaux (LAN)
    3. WAN : (Wide Area Network = réseau grande distance) : sont des réseaux étendus (large),et qui assurent la transmission des données sur des longues distances Il existe d’autres comme: PAN , WPAN , SAN

  • L’ouverture (leur utilisation) :

    1. Intranet : le réseau interne d’une entité organisationnelle ou d’une entreprise.
    2. Extranet : le réseau externe d’une entité organisationnelle ouvert vers l’extérieur.
    3. Internet Les réseaux publics

  • Les supports de connexion :

    1. Réseaux filaires
    2. Réseaux sans fils

  • Le type d’organisation :

    1. Les réseaux Postes à Postes (Peer to Peer, ou d’égal à égal)
    2. Les réseaux Clients/Serveurs: La plupart des stations sont des « postes clients »

6/ Le type de topologie :

  • Les réseaux en bus (ETHERNET, Token-Bus...) : Les machines sont reliées par un câble coaxial (le bus) , et chaque ordinateur est connecté en série sur le bus .
  • Les réseaux en étoile (téléphonie, STARLAN...) : Toutes les stations sont connectées à un commutateur, (un nœud central). SWITCH .
  • Les réseaux en anneau (FDDI, Token-Ring):: Les ordinateurs sont situés sur une boucle et communiquent chacun à leur tour. Elle utilise la méthode d'accès à "jeton" Token ring
  • Maillée (ATM, Réseaux MAN et WAN surtout) :Les réseaux maillés utilisent plusieurs chemins de transferts entre les différents nœuds.

Chapitre 3: Le modèle OSI

cette section a été faite par ouaheb beahadin et saadi zaid

Les Composants matériel d’un Réseau informatique:

  • les cartes réseau: également appelées cartes d'interface réseau, qui sont des composants matériels permettant à un ordinateur de se connecter à un réseau tel qu'Internet ou un réseau local.

    • Les cartes réseau jouent un rôle essentiel dans la communication entre ordinateurs sur un réseau en permettant à l'ordinateur de transmettre et de recevoir des données via des protocoles de communication spécifiques. également deux types de composants matériels utilisés pour connecter des appareils sur un réseau local : les switches et les hubs.

  • Les switches: permettent à plusieurs appareils de communiquer entre eux en utilisant des adresses MAC et utilisent une table de correspondance pour envoyer les données uniquement à l'appareil destinataire.
  • Les hubs: fonctionnent comme des répéteurs, en recevant les données envoyées par un appareil et les transmettant à tous les autres appareils connectés, mais ne possèdent pas la fonctionnalité de table de correspondance.

    • Les switches sont plus efficaces pour la gestion du trafic réseau et permettent une communication plus rapide et plus sécurisée entre les appareils connectés, tandis que les hubs sont encore utilisés dans des applications spécifiques ou dans des réseaux de petite taille où la complexité d'un switch n'est pas nécessaire.

2/ Le Modèle OSI:

Le modèle OSI est un modèle de référence pour les communications de réseau qui divise les communications en sept couches distinctes. Chaque couche est responsable de fonctions spécifiques liées à la communication de données entre les appareils de réseau. Les sept couches du modèle OSI sont:

  1. la couche physique
  2. la couche liaison de données
  3. la couche réseau
  4. la couche transport
  5. la couche session
  6. la couche présentation
  7. la couche application.

3/ Le Modèle TCP/IP

Ce modèle est souvent utilisé comme référence pour la conception et la mise en œuvre de réseaux informatiques. Bien qu'il ne soit pas utilisé dans les protocoles de communication de réseau modernes tels que TCP/IP, il reste utile pour comprendre les principes fondamentaux de la communication de réseau. Le modèle TCP/IP est un modèle de référence pour les communications de réseau. Il divise les communications en quatre couches principales :

  1. la couche liaison de données
  2. la couche réseau
  3. la couche transport
  4. la couche application

La couche liaison de données s'occupe de la transmission des données sur le support physique, la couche réseau assure le routage des données, la couche transport gère l'acheminement des données de bout en bout, et la couche application permet la communication entre les applications sur les appareils de réseau. Le modèle TCP/IP est utilisé dans la conception et la mise en œuvre des réseaux informatiques modernes, tels que les réseaux d'entreprise, les réseaux Internet et les réseaux sans fil. Il est basé sur les protocoles TCP, UDP et IP, et est largement utilisé dans les communications Internet.

4/ L’adressage:

Un adressage IP est un système utilisé pour identifier et localiser des dispositifs connectés à un réseau informatique, tels que des ordinateurs, des serveurs, des imprimantes et des appareils mobiles. Chaque appareil connecté à un réseau dispose d'une adresse IP unique, qui est une combinaison de chiffres séparés par des points (par exemple, 192.168.0.1). L'adressage IP permet aux appareils de communiquer entre eux en identifiant de manière unique l'expéditeur et le destinataire des données échangées sur le réseau. C’est quoi un adressage IP : Il existe deux principales versions d'adresses IP : IPv4 (utilisée depuis longtemps) et IPv6 (une version plus récente qui permet plus d'adresses disponibles pour répondre aux besoins croissants d'Internet).

L'adresse IPv4 :

IPv4 (Internet Protocol version 4) est la quatrième version du protocole Internet, qui est largement utilisée depuis son développement au début des années 1980. Les adresses IPv4 sont des nombres binaires de 32 bits, généralement représentés sous une forme décimale pointée composée de quatre ensembles de nombres décimaux séparés par des points (par exemple, 192.168.0.1).

Voici quelques points clés concernant IPv4 :

  • Format d'adresse : Les adresses IPv4 sont composées de deux parties : une partie réseau et une partie hôte. La partie réseau identifie le réseau auquel un appareil appartient, tandis que la partie hôte identifie l'appareil spécifique sur ce réseau.
  • Espace d'adressage : IPv4 offre environ 4,3 milliards (2^32) d'adresses uniques. Cependant, en raison de la croissance exponentielle d'Internet, les adresses IPv4 sont devenues rares.

Classes et sous-réseaux :

  • IPv4 définissait à l'origine trois classes d'adresses : la classe A, la classe B et la classe C. Ces classes avaient différentes plages de bits de réseau et d'hôte, ce qui permettait de créer des réseaux de tailles différentes
  • Adresses IP privées : IPv4 réserve certaines plages d'adresses pour les réseaux privés, qui ne sont pas routables sur Internet public.

Traduction d'adresse réseau (NAT) :

Le NAT est une technique utilisée pour surmonter la limitation des adresses IPv4 disponibles. Il permet à plusieurs appareils au sein d'un réseau privé de partager une seule adresse IP publique. Le NAT traduit les adresses IP privées des appareils en adresse IP publique lors des communications avec Internet.

  • Structure des paquets IPv4 : Les paquets IPv4 sont composés d'un en-tête et d'une charge utile. L'en-tête contient des informations telles que les adresses IP source et destination, la longueur du paquet, le protocole et le temps de vie (TTL). La charge utile transporte les données réellement transmises.

Chapitre 5: Système de Gestion de Version

cette section a été faite par abdelhamid toubache aissani salah eddine

Ce chapitre constitue une introduction à Git et GitHub, les deux systèmes de gestion de version les plus populaires parmi les développeurs. Voici un résumé des points clés abordés dans chaque section :

  1. Introduction :

    Ce chapitre explore l'importance de Git et GitHub dans le développement de logiciels et définit leur utilité. Git, un outil de gestion de versions créé par Linus Torvalds en 2005, permet de conserver un historique détaillé des modifications apportées à un projet. En utilisant des logiciels de gestion de versions, les équipes peuvent collaborer de manière plus efficace et améliorer leur productivité.

  2. L'utilité concrète d'un système de gestion de version :

    Dans cette section, nous explorons l'importance d'un système de gestion de versions en utilisant un site web comme exemple. Lorsque plusieurs développeurs travaillent sur un projet commun, il devient crucial de coordonner leurs modifications, d'éviter les conflits et de garantir que chaque développeur dispose de la version la plus récente du projet. Un système de gestion de versions offre une solution efficace pour gérer ces aspects de manière optimale.

  3. Les deux modèles des logiciels de gestion de version :

    Dans cette section, nous abordons les deux modèles de gestion de versions les plus répandus : le modèle centralisé et le modèle décentralisé. Dans le modèle centralisé, le code source est stocké sur un serveur central, tandis que dans le modèle décentralisé, chaque utilisateur possède une copie complète du code source sur sa propre machine. Git adopte le modèle décentralisé, ce qui confère une plus grande flexibilité et une meilleure sécurité.

  4. Qu'est-ce que GitHub ?

    Git est un logiciel de gestion de version extrêmement puissant, tandis que GitHub, quant à lui, est un service en ligne renommé qui offre la possibilité d'héberger des dépôts Git. En réalité, GitHub se distingue en tant que premier et plus vaste hébergeur de dépôts Git à l'échelle mondiale. Les développeurs, quant à eux, peuvent aisément partager leur code sur cette plateforme et collaborer avec d'autres en proposant et en ajoutant des fonctionnalités supplémentaires. Grâce à cette combinaison de Git et de GitHub, les équipes de développement ont l'avantage de bénéficier d'un environnement de travail plus efficace et productif pour leurs projets en commun.

  5. L'utilisation de Git :

    Dans cette section, diverses interfaces pour utiliser Git sont présentées, avec un accent particulier sur l'utilisation de la ligne de commande. L'utilisation de la ligne de commande permet d'accéder à toutes les commandes Git disponibles et assure une expérience cohérente pour tous les utilisateurs, quel que soit leur système d'exploitation ou leur environnement de développement.

  6. Installation de Git:

    Les étapes d'installation de Git sont expliquées, recommandant de télécharger la dernière version à partir du site officiel et de suivre les instructions d'installation par défaut.

  7. Paramétrage de Git:

    Après l'installation, il est nécessaire de configurer Git en fournissant un nom d'utilisateur et une adresse e-mail. Cela peut être fait en utilisant la commande "git config" avec l'option "--global" pour appliquer les informations de configuration à tous les projets Git.

  8. Démarrer un dépôt Git:

    Un dépôt Git est la copie et l'importation de l'ensemble des fichiers d'un projet dans Git. Il existe deux façons de créer un dépôt Git : importer un répertoire existant ou cloner un dépôt déjà existant. Les étapes pour démarrer un dépôt Git sont expliquées.

  9. La gestion des informations selon Git:

    Git gère les informations en prenant des instantanés du contenu de l'espace de travail à chaque validation.

  10. Les états des fichiers:

    Les états des fichiers dans Git sont gérés pour suivre les modifications sur les fichiers d'un projet. Un fichier peut être soit "sous suivi de version" (tracked), ce qui signifie qu'il fait partie du dernier instantané enregistré par Git, soit "non suivi" (untracked), ce qui indique qu'il ne fait pas partie du dernier instantané et n'a pas été indexé.

  11. Les zones du travaille :

    Ces états de fichiers sont associés à trois zones de travail dans Git :

    • Le répertoire de travail correspond à l'extraction des fichiers d'une version spécifique du projet, où l'on peut les utiliser et les modifier.
    • La zone d'index est un fichier situé généralement dans le répertoire Git, qui stocke les informations sur les fichiers qui seront inclus dans le prochain instantané.
    • Le répertoire Git est l'endroit où Git stocke les méta-données et la base de données des objets du projet.

  12. Ajouter ou modifier des fichiers dans un projet et actualiser notre dépôt Git :

    Le processus de travail dans Git consiste à modifier les fichiers dans le répertoire de travail, puis à les indexer en utilisant la commande "git add". Une fois les fichiers indexés, ils peuvent être validés en utilisant la commande "git commit", ce qui les ajoute au répertoire Git.
